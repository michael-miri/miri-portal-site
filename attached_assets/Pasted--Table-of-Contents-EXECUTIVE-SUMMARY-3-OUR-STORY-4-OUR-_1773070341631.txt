System Overview and Architecture

Platform Overview
Miri | Time and Attendance is a platform designed to support distributed workforces and modern workforce management requirements. The system enables employees to record attendance through a responsive browser-based interface accessible from desktop and mobile devices. Because the platform is fully browser-based and responsive, staff can securely access the system from any modern web browser without requiring specialized software installation.
The platform supports configurable workflows, reporting, and operational rules, allowing administrators to tailor the system to the Office of the State Auditor’s operational requirements. This configurability enables OSA administrators to manage work locations, approval processes, and reporting structures through system configuration rather than software customization.

System Architecture
The Miri platform is built using a modern cloud-based full-stack architecture designed to provide reliability, scalability, and secure system operation. The application is delivered through a responsive web interface built using React, allowing users to access the system through any modern web browser on desktop or mobile devices.
The application logic is supported by a Node.js-based API layer, which manages system workflows, user authentication, data processing, and communication between application components. This API architecture allows the platform to support secure integrations with external systems while maintaining consistent system performance.
Application data is stored within a PostgreSQL relational database hosted through Amazon Relational Database Service (Amazon RDS). This managed database service provides automated backups, system monitoring, and reliable data storage capabilities while reducing administrative overhead.

Cloud Infrastructure
The Miri platform is hosted in the AWS US East (Ohio) region, ensuring that all application infrastructure and data storage remain within the United States. Hosting the platform in AWS provides a highly available and scalable infrastructure capable of supporting OSA’s workforce management needs.
AWS provides enterprise-grade infrastructure services including automated system monitoring, infrastructure redundancy, and secure data storage. These capabilities allow Miri to deliver a reliable cloud-based platform while minimizing operational complexity for OSA.

Security Architecture
Security is a core design component of the Miri platform. The system leverages AWS security services and industry-standard security practices to protect sensitive data and ensure system integrity.
Data transmitted between users and the platform is encrypted using Transport Layer Security (TLS). Data stored within the system is encrypted at rest using AES-256 encryption through AWS Key Management Service (KMS).
Application infrastructure is deployed within an AWS Virtual Private Cloud (VPC) environment that restricts network access and isolates system components. Database services hosted through Amazon RDS provide automated backups and recovery capabilities to ensure data durability and resilience.

Authentication and Identity Management
The Miri platform supports Enterprise Single Sign-On (SSO) authentication, allowing the Office of the State Auditor to integrate the system with its existing identity management environment. Through SSO integration, staff can securely access the platform using their existing organizational credentials.
This approach simplifies user access management while maintaining centralized control over authentication policies and account management.

Performance and Scalability
The Miri platform is designed to support the operational needs of the Office of the State Auditor and can scale to support the required number of system users. The cloud-based architecture allows the system to accommodate expected usage levels, including concurrent access during peak clock-in and clock-out periods.
Because the platform is hosted in AWS, system resources can be scaled as needed to support future growth while maintaining consistent application performance.

Integration Approach
Integration Overview
The Miri platform provides a secure and flexible integration framework designed to support data exchange with existing enterprise systems used by the Office of the State Auditor. The platform supports modern API-based integration as well as secure file-based data exchange where direct API connectivity is not available.
This approach ensures reliable communication between systems while allowing integration methods to adapt to the technical capabilities of external systems.
API-Based Integrations
Miri supports integration with external systems using REST-based APIs over secure HTTPS connections. API integrations allow systems to exchange data in a structured and efficient manner while maintaining strong security controls.
API authentication is supported through OAuth 2.0 or secure token-based authentication mechanisms, ensuring that system-to-system communication occurs through authenticated and encrypted channels.
This architecture allows Miri to integrate with enterprise systems in a secure and scalable manner.
File-Based Integration
In situations where API connectivity is not available or supported, the Miri platform can support secure file-based data exchange as an alternative integration method.
File-based integrations may utilize structured data formats and secure transfer methods to exchange system data. This approach ensures that integration can still be achieved even when external systems do not support direct API connectivity.
HRCMS Integration
The Miri platform supports integration with the Commonwealth Human Resources/Compensation Management System (HRCMS) through secure API-based communication where available.
This integration allows relevant workforce data to be exchanged between systems to support workforce management processes and maintain alignment with the Commonwealth’s HR systems.
Where API connectivity is not available, secure file-based data exchange methods can be used to support data synchronization.
Wolters Kluwer TeamMate+ Integration
The platform also supports integration with Wolters Kluwer TeamMate+, enabling data exchange between the time and attendance system and the audit management platform.
This integration allows operational data to be shared between systems in order to support the agency’s audit and investigative workflows.
Integration may be implemented using REST-based APIs or secure file-based exchange depending on the capabilities of the external system.
Data Export Capabilities
The Miri platform supports export of system data in common formats to support reporting and external system integration. Supported export formats include:
•	CSV
•	Microsoft Excel
•	PDF
These export capabilities allow administrators to generate reports or transfer data to other systems when necessary.
